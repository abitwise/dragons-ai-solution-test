## Cross-Phase Integration

Integration checker (gsd-integration-checker) traced the full dependency chain and E2E flows:

- **Dependency graph (verified, acyclic):** `types.ts` (leaf) ← `decode.ts` / `fake-api-client.ts` / `api.ts` ← `strategy.ts` (types-only) ← `runner.ts` ← `logger.ts` / `index.ts`.
- **Seam discipline verified:** `api.ts` is the only `fetch()` caller; `strategy.ts` imports only `./types.js`; `index.ts` is the only production construction site for `HttpApiClient` + `createConsoleLogger` and the only `playGame` caller.
- **`END` single source of truth:** defined in `runner.ts`, consumed by `index.ts` exit-code mapping; a compile-time assertion (`_AssertEndValuesAreEndReason`) fails `tsc` if the reason strings drift from the `EndReason` union.
- **E2E Flow 1 (LOOP-01):** `index.main()` → construct client+logger → `playGame` → per-turn (getShop/buy → getMessages → chooseAd → solve → state merge) → `GameReport` → `printBanner` + exit code. WIRED.

## Tech Debt (non-blocking, advisory)

All items below are documented advisory quality findings from phase code reviews. None
block the milestone; the highest-risk one (Phase 1 WR-01 non-finite reward) was already
hardened downstream in Phase 2 with `Number.isFinite` guards in `strategy.ts`.

**Phase 1 (foundation):**
- WR-01: `z.coerce.number()` maps empty/null `reward` to `0` rather than rejecting as schema drift (mitigated in Phase 2).
- WR-02: empty `gameId`/`adId`/`itemId` yields a malformed double-slash URL with no early guard (callers always pass a valid `gameId`).
- WR-03 / IN-01..IN-04: minor smells — redundant Base64 length check, no retry jitter, double-matched test glob in `tsconfig.include`.
- WR-04: `FakeApiClient` function-source path does not `await` a returned Promise (test double only; all sources currently synchronous).

**Phase 2 (strategy):**
- WR-01/WR-03 rationale comments are factually inaccurate (the zod boundary already rejects non-finite values) — the guards themselves are correct defense-in-depth.
- Theoretical EV overflow (`reward × rank → Infinity`) is not reachable via the live Mugloar API.
- IN-01: stale plan-progress comment in `strategy.ts` omits the 02-05 gap-closure pass.

**Phases 3 & 4:** no anti-patterns or debt markers found.
