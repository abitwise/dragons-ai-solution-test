<objective>
Drive the ad-selection core test-first (STRAT-03): `chooseAd` picks the eligible ad with the highest expected value (`reward x rank`), breaks ties by sooner expiry then higher reward, falls back to a least-bad gamble when no ad clears the floor, and returns an explicit "no ad" signal (`null`) only when the board is truly empty — never throwing, never selecting a still-encrypted ad.

Purpose: Selection IS the product — picking which ad to solve is the heart of the bot's score. Building it test-first against a mixed board locks in the EV-over-raw-reward behavior (PITFALLS #4: reward and risk are positively correlated) and the deterministic tiebreak that keeps tests stable. The `null` no-ad signal (D-07) is the contract the Phase 3 runner branches on to shop or end cleanly.

Output: `chooseAd` in `src/strategy.ts` (reusing Plan 01's rank table and eligibility filter); tests in `src/strategy.test.ts` covering mixed-board EV, the expiry-aware tiebreak, the least-bad-gamble fallback, and the empty-board null signal.
</objective>
